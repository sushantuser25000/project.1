# File Overview

This document provides a detailed breakdown of the files in the project and their respective roles.

## Root Directory
- `OrgRegistry.sol`: The smart contract for registering and authorizing organizations (verifiers). It handles verification requests, audit trails, and organization status.
- `UserAuth.sol`: The core smart contract handling user registration, document metadata storage, and on-chain verification lookups (`verifyById`, `verifyByHash`).
- `README.md`: Basic project introduction.
- `PROJECT_OVERVIEW.md`: High-level summary of the system.
- `SETUP_GUIDE.md`: Step-by-step instructions for environment setup.
- `QUICK_COMMANDS.md`: Reference for common terminal commands.
- `genesis.json`: Configuration for the custom private Geth blockchain.
- `start-node1-updated.bat`: Script to launch the Geth blockchain node.

## Backend (`/backend`)
- `server.js`: The central Express API server. It bridges the frontend with the blockchain and Google Drive.
- `drive_service.js`: Integration logic for Google Drive API (upload, download, encryption, and folder management).
- `.env`: Secret configuration (RPC URLs, Contract Addresses, Private Keys). **Never share this file.**
- `pending_docs.json`: Local database track documents while they await on-chain verification.
- `list_users.js`: Utility script to view all registered users on the blockchain.
- `acem_key.txt`: (Manual Note) Stores the generated private key for the ACEM Admin organization.

## Frontend (`/frontend`)
- `src/components/`:
    - `VerifyPage.jsx`: The main public verification UI where users enter IDs or upload files to check authenticity.
    - `AdminDashboard.jsx`: Interface for ACEM Admins to manage organizations and verifications.
    - `UploadPage.jsx`: Where registered users upload documents to the system.
- `src/context/AuthContext.jsx`: Manages global user authentication state.
- `.env`: Frontend-specific environment variables (API URLs, Contract Addresses).

## Hardhat (`/hardhat-example`)
- `contracts/`: Source code for smart contracts.
- `hardhat.config.js`: Configuration for the Hardhat development and testing framework.
- `deploy-vanilla.js`: Script used to deploy the smart contracts to the Geth node.
- `artifacts/`: Compiled contract files (ABI and Bytecode) generated by Hardhat.

---

## File Recovery (Git)
The project uses **Git** for version control. These files are permanent records.

- **Wait, where is `.env`?** `.env` files are ignored by Git for security (they contain sensitive keys).
- **How to recover?** If you accidentally delete a file like `server.js`, you can recover it using Git:
    - Run `git status` to see what changed.
    - Run `git checkout server.js` to restore the file to its last saved state.
- **Accidental Deletion of `.env`?** These files are NOT tracked by Git. If you delete them, they cannot be recovered by Git. **Keep a backup of your contract addresses and private keys separately.**
